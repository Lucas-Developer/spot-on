2015.06.19
  1. Acceptable URL schemes: ftp, gopher, http, https.
  2. Added libspoton_delete_urls() to libspoton. Version updated to 0.1.1.
  3. Allow adjustment of the iteration count in the file encryption tool.
  4. Allow configuration of alternating row colors of Chat, E-Mail, and URLs participants. Please see Options.
  5. Allow configuration of the kernel's cache purge timer.
  6. Allow importing of URLs without Import Credentials. Only plaintext URLs will be imported.
  7. Assume zero pointer defaults in Qt::value().
  8. Bound size_t parameter of memcpy() in spoton_crypt::setHashKey(), spoton_misc::peerAddressAndPort(), spoton_sctp_server::setMaxPendingConnections(), spoton_sctp_socket::localAddressAndPort().
  9. Close progress dialogs instead of hiding them. Please see https://en.wikipedia.org/wiki/Shroedinger%27s_Cat.
  10. Completed linear URL distribution. It is expected that Common Credentials are created. Items that cannot be imported will be discarded.
  11. Compute automatic digests in threads. Safely interrupt on process termination. Completed futures are occasionally removed.
  12. Corrected importing of public keys. Cannot assume that key pairs exist.
  13. Corrected re-encoding of listeners_accounts. Must include the listener_oid field in the where-clauses.
  14. Corrected re-encoding of listeners_allowed_ips. Must include the listener_oid field in the where-clauses.
  15. Corrected re-encoding of transmitted_magnets. Must include the transmitted_oid field in the where-clauses.
  16. Corrected removal of URLs having special characters.
  17. Corrected setting of remote IP information. Please see Neighbors.
  18. Corrected use of spoton::addFriendsKey().
  19. Custom status messages. A maximum of 64 characters will be imposed.
  20. Decreased message size of Rosetta output.
  21. Denote extreme time deltas in Chat. Currently configured to thirty seconds. Source only.
  22. Derive subsequent file encryption key from initially-derived key via SHA-256 (Key_(i + 1) = SHA-256(Key_i)).
  23. Digital signatures should vouch for private key information.
  24. Display URL database connection information within the Search panel.
  25. Do not assume that Poptastic values have been accepted by the user. That is, query the necessary database in order to retrieve required values.
  26. Do not delete URLs from shared.db unless they are properly loaded.
  28. Do not initiate a rapid StarBeam if the specified participant is temporary.
  29. Do not distribute private spoton_crypt objects to child objects as the contents of the spoton_crypt objects may be modified by the re-encoding process. Instead, use spoton::instance()->crypts().
  30. Dual-purpose domain check boxes. Please see URLs.
  31. Encapsulate data into streams and decrease Base-64 conversions. This will result in smaller messages. Previous functionality will be disrupted.
  32. Establish authentication and encryption keys via silent SMP. The authentication and encryption keys are derived from a Whirlpool digest of the secret.
  33. Guard QList::last().
  34. Guard QList::removeAt().
  35. Impose a maximum number of keywords in received URLs. Currently set to 100.
  36. Increased congestion cost maximum to 65536.
  37. Increased default secure memory pool sizes to 262144 bytes. Increased minimum pool sizes to 131072.
  38. Inspect results of all QDataStream operations.
  39. Log re-encoding errors.
  40. Many integer conversion corrections.
  41. McEliece public-key encryption. Incomplete.
  42. Modified QFileInfo::lastModified() usage. It's possible that a less-than-or-equal to comparison will leave containers in unpleasant states.
  43. Notify the user of URL import statistics.
  44. Per Mr. Schmidt, allow distribution of public key pairs via established secrets.
  45. Please remove urls_distillers_information.db before launching Spot-On.
  46. Populate URL distillers before performing an import.
  47. PostgreSQL administrators must recreate Spot-On databases.
  48. Prepare URL cipher and hash widgets if libgcrypt fails.
  49. Prompt before accessing a remote URL.
  50. Re-encode URL common and import credentials information.
  51. Removed GCRY_MD_FLAG_SECURE from all gcry_md_open() instances.
  52. Removed duplicate QObject::connect() instance from spoton::slotConfigurePoptastic().
  53. Removed input masks from IPv6 fields.
  54. Replace spaces with dashes in StarBeam and e-mail attachment file names.
  55. Replaced (rv == 0) with (rv == SQLITE_OK) in libspoton.c.
  56. Replaced the itemClicked() signal on the e-mail table with itemDoubleClicked() as itemClicked() and itemSelectionChanged() may cause duplicate dialog prompts.
  57. Require account salts to contain at least 512 bytes.
  58. Require the messages in 0002a and 0002b to contain at least 64 bytes.
  59. Simplified copying and exporting of public keys.
  60. The method QTableWidget::findItems() may return items that are irrelevant to the query. Implemented a special column-based method.
  61. Updated NTRU files (05/31/2015).
  62. Upgraded OpenSSL packages to version 1.0.2c on Windows.
  63. Upgraded Qt packages to version 4.8.7 on OS X and 5.4.2 on Windows.
  64. Upgraded SQLite to version 3.8.10.2 on Windows.
  65. Verify a StarBeam's provided pulse size against some defined maximum.
  66. Verify results of ntru_rand_init(). Log errors.

  13. On Qt 5.x and Windows, assume a fusion style.
  14. QWidget::activateWindow() should only be used on Windows!
  15. Updated SQLite to version 3.8.9, Windows.
